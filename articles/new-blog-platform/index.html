New Blog Platform

26 Jan 2013

I'm one of those bloggers that change their blog platform more often than is really required. I'm also one of the bloggers that don't blog very often - and when I do, the posts usually start with how I'm going to do it more often. This time, however, I'll just talk about what I'm using now and why.

When I switched to my previous platform, Octopress, I was excited to be using a static site generator. I agree with Mick Gardner on the virtues of static site generators. I also liked the ability to author them in markdown, use source control for the posts, and publish to GitHub Pages. At the time I was really enjoying writing code in Ruby.

While I still think Ruby is a fantastic language, my infatuation with Node.js overcame me a couple of years ago and it's been a beautiful affair ever since. Becuase I'm not a regular blogger, there wasn't a rush to change platforms, despite the itch. During one of my scratching sessions, I ran across Boris Mann's post on Node Based Static Site Generators. I had known about Wheat for a long time, and spent some time evaluating Blacksmith, which I found a little to incomplete for my needs, DocPad, which I found to be heavier than I needed for my blog, but will likely use it for other things, and Wintersmith, which I decided on.

Since my (only 3) existing posts were already in markdown, moving to the new generator was a snap. The only thing that was time consuming was applying the new design, which I wanted anyway. It's not done by the way, but I want to finish the upcoming easter egg before refining the design anymore.

Thanks Johan. I'd like to eventually have a draft feature added. I'll send the pull request when I get around to it :)
